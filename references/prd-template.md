# PRD Template

Use this template when outputting a PRD in Stage 1.5.
Read this file before generating the PRD block.

## PRD Output Format

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 PRD · [项目名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

| 项目 | 内容 |
|---|---|
| 输出类型 | 网页 / 报告 / PPT |
| 一句话定位 | 用一句话说清楚这是什么、给谁用、有什么不同 |
| 核心用户 | 具体的人，不是"所有人"——说出他们的状态或行为特征 |
| 核心场景 | 最多 3 条，每条格式：用户在 [状态] 时，打开这个页面，希望 [得到什么] |
| 页面唯一任务 | 这个页面只负责一件事，用一句话说 |
| 必须有 | 列出不可缺少的内容或功能 |
| 明确不做 | 列出不包含的内容，防止范围蔓延 |
| 成功标准 | 用户做了什么行为才算这个页面成功了 |

## 填写规则

### 一句话定位的写法

不好的写法：
- "一个设计工具"
- "帮助用户做设计的平台"

好的写法：
- "给不会写代码的设计师用的 AI 原型生成工具，10 秒出一个可点击的 demo"
- "只做一件事的灵感启动页——打开后 5 秒内给用户一个可以开始做的方向"

### 核心场景的写法

不好的写法：
- "用户想做设计"

好的写法：
- "用户今天想开始一个小项目，但脑子空白，打开这个页面希望被一个句子触发想法"
- "用户有一个模糊的需求，打开页面后希望通过换点子找到自己真正想做的方向"

### 成功标准的写法

不好的写法：
- "用户满意"
- "页面好看"

好的写法：
- "用户点了'换一个点子'超过 3 次"
- "用户复制了某条想法并去做了"
- "用户在 10 秒内产生了一个想法，不需要继续换"

## PRD 完成后的操作

输出 PRD 后，问：
```
这份 PRD 准确吗？有要调整的地方吗？
确认后进入视觉方向选择。
```

如果用户修改了 PRD，只更新被修改的部分，保留其他已确认的内容。

PRD 确认后，更新 session state 中的 `prd_confirmed: true`，然后进入 Stage 2（参考图摄入）。

## 不同输出类型的 PRD 差异

### 网页类型

重点在：
- 首屏传达什么（一句话）
- 用户应该做的唯一行动是什么（CTA）
- 不在首屏的内容怎么安排

### 报告类型

重点在：
- 报告的核心论点是什么
- 目标读者的知识背景
- 需要哪些章节，每章的核心结论
- 是否需要打印、引用、目录跳转

### PPT 类型

重点在：
- 演讲场景（几人？线上还是线下？正式还是非正式？）
- 幻灯片数量和时长
- 每张片的核心是什么（一个关键点 + 一个视觉）
- 是否需要演讲者备注
