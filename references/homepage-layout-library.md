# Homepage Layout Library

Use this reference when the user cares about homepage originality, hero variety, or when
multiple generated pages are starting to look too similar.

## Why Homepages Start Looking The Same

Many generated pages collapse into the same hero archetype:

- top navigation
- left text block
- right image or product visual
- eyebrow above headline
- two CTAs under the copy
- proof or utility strip below

That archetype is not wrong.
It is only one valid option.
The problem starts when it becomes the default for unrelated brands, products, and moods.

## Hero Diversity Rule

When designing a homepage, explicitly choose a named hero archetype.

Do not silently default to a split hero.

If generating multiple homepage concepts or multiple websites in one session:

- do not reuse the same archetype twice in a row unless the user asks for continuity
- if a previous output already used a left-copy / right-media split hero, prefer a
  different hero posture next time
- vary at least four of these dimensions:
  - copy position
  - media position
  - proof position
  - CTA position
  - headline line-break style
  - navigation posture
  - hero height
  - section transition style

## 1. Split Editorial Stage

### Structure

```text
| nav |
| copy | media |
| CTA row |
| support strip |
```

### Best For

- portfolios
- product pages with one strong visual
- music / fashion / camera / creator brands

### Strength

Easy to read and easy to convert.

### Risk

This is the most overused AI hero archetype.

### Distinguishing Moves

- make the split asymmetric rather than 50/50
- move proof into the copy column instead of below
- use unusual headline stacking or vertical rhythm
- avoid generic rounded SaaS cards

## 2. Full-Bleed Poster Hero

### Structure

```text
| nav overlay |
| big statement on image/video |
| compact CTA cluster |
| minimal metadata |
```

### Best For

- artists
- campaigns
- fashion
- entertainment
- event landing pages

### Strength

Immediate mood and memorability.

### Risk

If the image is weak, the whole page weakens.

### Distinguishing Moves

- treat type like poster typography, not UI copy
- keep nav and CTA small
- let the visual occupy most of the first screen

## 3. Manifesto Stack

### Structure

```text
| slim nav |
| huge thesis text |
| short supporting paragraph |
| single CTA |
| proof / logos / numbers |
```

### Best For

- design-led tools
- AI products that need trust
- consulting / studio sites

### Strength

Makes the message feel authored and confident.

### Risk

Needs strong copy discipline.

### Distinguishing Moves

- no large hero image required
- typography does the heavy lifting
- proof appears earlier than features

## 4. Product Frame Hero

### Structure

```text
| nav |
| product window / device frame centered or offset |
| short headline |
| utility CTA row |
| quick proof metrics |
```

### Best For

- software
- dashboards
- apps
- tools with strong UI

### Strength

Shows the product immediately.

### Risk

Can become generic SaaS if the frame is not art-directed.

### Distinguishing Moves

- use a real product moment, not a fake dashboard collage
- let annotations explain one key moment
- keep copy tighter than a standard marketing hero

## 5. Gallery Mosaic Hero

### Structure

```text
| nav |
| headline block |
| mosaic of 3-7 visual tiles |
| CTA tucked into grid or margin |
```

### Best For

- portfolios
- case study hubs
- music / film / photography
- multi-product brands

### Strength

Feels alive and less templated.

### Risk

Can get noisy if all tiles compete equally.

### Distinguishing Moves

- choose one anchor tile and make others support it
- use mismatch in tile scale intentionally
- avoid turning every tile into the same card

## 6. Narrative Scroll Teaser

### Structure

```text
| nav |
| short opening statement |
| partial next section visible |
| scroll cue / chapter markers |
```

### Best For

- storytelling brands
- editorial launches
- high-concept studio sites
- immersive product narratives

### Strength

The hero and second section feel like one continuous experience.

### Risk

Needs better rhythm control than a standard hero.

### Distinguishing Moves

- reveal the next chapter in the first screen
- let the first scroll be part of the hero
- use chapter framing or rail markers

## 7. Proof-First Command Hero

### Structure

```text
| nav |
| headline + proof stats side by side |
| trust row |
| CTA after proof |
```

### Best For

- B2B
- AI infrastructure
- enterprise tools
- compliance-heavy products

### Strength

Trust appears before aesthetic flourish.

### Risk

Can feel dry without one memorable visual move.

### Distinguishing Moves

- use numbers, certifications, or benchmark snippets early
- keep visuals diagrammatic and 2D
- let proof be part of the hero, not a later section

## 8. Docked Interface Hero

### Structure

```text
| nav |
| core visual anchored center |
| secondary controls or playlist docked on edge |
| persistent utility band |
```

### Best For

- music
- media
- creator tools
- interactive brand sites

### Strength

Feels productized and specific.

### Risk

Too many controls can crowd the hero.

### Distinguishing Moves

- treat the dock as a character, not just chrome
- let one control cluster stay persistent
- use the dock to imply interactivity even in static layout

## Quick Selection Matrix

| Archetype | Visual weight | Copy weight | Best when | Avoid when |
| --- | --- | --- | --- | --- |
| Split Editorial Stage | medium-high | high | one strong visual + clear CTA | you already used this once |
| Full-Bleed Poster | very high | medium | mood is the product | visuals are weak |
| Manifesto Stack | low-medium | very high | message must lead | copy is generic |
| Product Frame | high | medium | product UI matters | product UI is unfinished |
| Gallery Mosaic | very high | medium | many assets exist | hierarchy is unclear |
| Narrative Scroll | medium | high | scroll experience matters | page is purely utilitarian |
| Proof-First Command | medium | high | trust must come first | brand needs spectacle |
| Docked Interface | high | medium | controls are part of the story | interaction model is unclear |

## Rotation Strategy

If a previous homepage used:

- `Split Editorial Stage`

then the next concept should prefer one of:

- `Full-Bleed Poster`
- `Manifesto Stack`
- `Gallery Mosaic`
- `Docked Interface`

If the previous homepage was already image-led, the next concept should either:

- become type-led
- become proof-led
- or become scroll-led

## Deliverable Pattern

When presenting homepage directions, explicitly include:

| Item | Value |
| --- | --- |
| Hero archetype | ... |
| Why this archetype fits | ... |
| What makes it different from previous concepts | ... |
| Transition into section 2 | ... |

This makes homepage structure visible and reduces accidental repetition.
